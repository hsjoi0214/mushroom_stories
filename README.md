
# A BORING MUSHROOM WEBSITE

This user-friendly, responsive website showcases a variety of Gourmet mushrooms, ensuring optimal viewing experience across an array of devices.

## User Experience (UX)

###Initial Discussion 

Embark on a culinary adventure with our informative website, featuring an extensive showcase of 16 exquisite Gourmet mushrooms! Each variety is presented with thier pics in their wild habitat along with some insightful details, igniting your curiosity.

key information:

1. Explore the diverse range of gourmet mushrooms, each boasting unique textures, tastes.
2. Delve into fascinating facts about these remarkable organisms, from their places of origins to their surprising health benefits.
3. We invite passionate mushroom enthusiasts to engage with us! Share your feedback and help shape the future of this resource.
4. Our website is a living repository of knowledge, regularly updated with the latest and most relevant information on the fascinating world of mushrooms.

###User stories

####Client Goals:

1. To be able to view the site on a range of device sizes.
2. Allow users to explore a diverse selection of 16 gourmet mushrooms and know interesting facts about them.
3. Users can leave feedback and contact the owner for further questions. 

####First Time Visitor Goals
1. Informing oneself with the different edible mushrooms.
2. As a first time user, to be able to navigate thru the website easily and to find the related information easily.

####Returning Visitor Goals
1. To brush up on my knowledge of mushrooms.
2. To see if any more mushrooms are added in the catalouge.

###Features
The website is comprised of 5 pages, links to every other page can be found from the page you are in. 

The website is meant to work error free in the following screen resolutions:

<iphone SE , screen size: 375 x 667px>
<ipad mini, screen size: 768 x 1024px>
<ipad air, screen size: 820 x 1180px>
<ipad pro, screen size: 1024 x 1366px>

General design of the website:
1. Ever page apart from the "Home" page have a navigation bar at the top which allows users to easily navigate through the website. The top left portion of the website has the heading along with navigation bars. 
2. Each page has a footer with links to corresponding social media accounts. Icons of social media's are used instead of text for better design readibility. 

Website pages description:
1. Home Page.
On the home page, the user is welcomed with a clean user interface of the website where only the absoulte essential things are kept. 

Top most part contains the website logo and the company name. 5 hyperlinks are given to navigate to the needed page without any hassel. 

The active page is furthermore indicated via an underline and hover effect both on the link and the underline. Additionally all the hyperlinks are hover aware and made responsive to click. 

2. Gallery page.
The gallery page has the company logo on the top left corner of the screen followed by a navigation bar that is only active until the user is accessing the website via a device with min-width less then 768px. 

Furthermore, the gallery page has kept clean and to the point for users to check out the images of different varities of mushrooms. 

Every image has a caption that is additionally linked to the specific section of the Learn.html page to reach to the facts & figures about the mushroom that the user is looking. 

3. About page.
The about page follows the same header and navigation styles as the gallery page. It tells the story about our company and why we started this website. 

This page has a "Read more about our mission section which is at the bottom of the page hidden which is styled with a hover effect and can be accesed by clicking on it"

Accessing the website on a device with min width of 768px or greater enables the appearnce of a delightful picture which is otherwise hidden away in order to save screen space. 

4. Learn page.
The Learn page has all interesting facts and summary about different mushroom varities mentioned in the website. 

Every section is further linked to its corresponding picture in the gallery.html page. 

5. Contact page.
A contact page to contact us or to leave a feedback. 

User can enter their email addresses and choose their favorite mushrooms from the questionaire. 

6. Future Implementation.
The website would be further developed into a commercial website where users can place order and leave feedback and reviews on the website for other users to see. 

###Accessibility
Mindful practice of code writing has been implemented during this project. 

Use of semantic tags are used wherever possible. 

Every page is marked active when a user enters it, making it easier for user to know where they are. 

Descriptive alt attributes on images are used.

## Deployment and Local Development

###Deployment
Github Pages was used to deploy the live website. The instructions to achieve this are below:

1. Log in (or sign up) to Github.
2. Find the repository for this project, mushroom_stories.
3. Click on the Settings link.
4. Click on the Pages link in the left hand side navigation bar.
5. In the Source section, choose main from the drop down select branch menu. Select Root from the drop down select folder menu.
6. Click Save. Your live Github Pages site is now deployed at the URL shown.

###Local Development

####How to Fork
To fork the mushroom_stories repository:

1. Log in (or sign up) to Github.
2. Go to the repository for this project, hsjoi0214/mushroom_stories.
3. Click the Fork button in the top right corner.

####How to Clone
To clone the mushroom_stories repository:

1. Log in (or sign up) to GitHub.
2. Go to the repository for this project, hsjoi0214/mushroom_stories.
3. Click on the code button, select whether you would like to clone with HTTPS, SSH or GitHub CLI and copy the link shown.
4. Open the terminal in your code editor and change the current working directory to the location you want to use for the cloned directory.
5. Type 'git clone' into the terminal and then paste the link you copied in step 3. Press enter.

## Testing

Testing was ongoing throughout the entire build. I utilised Chrome developer tools while building to pinpoint and troubleshoot any issues as I went along.

The following issues were raised during my mid project meeting with my mentor:

1. The pages were not media responsive. This now have been corrected by using @media screen property in .css. 

2. The picture in the about page taking up much screen space and was making other elememts out of line. This is now corrected by applying display , height and width property to the picture.
